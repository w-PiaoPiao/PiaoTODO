# 开发日志

本目录用于记录 LiteToDo 的每日开发进展、完成事项、遇到的问题和下一步计划。

## 文件名规则

每个日志文件按日期命名：

```
YYYY-MM-DD.md
```

例如：`2026-06-29.md`

## 日志模板

每个日志文件包含以下部分：

```markdown
# 2026-06-29 开发日志

## 今日目标

- [ ] 目标 1
- [ ] 目标 2

## 完成事项

1. 完成事项 A
2. 完成事项 B

## 遇到的问题

- 问题 1：描述和解决方案

## 下一步计划

- 下一步 1
- 下一步 2

## Notes

- 临时想法、决策记录、参考资料等
```

## 使用方式

1. 每天开发开始前，基于前一天日志创建新日期的日志文件。
2. 开发过程中随时更新「完成事项」和「遇到的问题」。
3. 当天开发结束时，完善「下一步计划」和「Notes」。
4. 如果一天内有多个开发会话，在同一个日志文件中追加记录。

## 自动化

理想情况下，每次开发会话结束时，由 AI 代理根据实际完成的工作自动更新日志。最低要求是每次完成一个 Phase 后手动记录。
